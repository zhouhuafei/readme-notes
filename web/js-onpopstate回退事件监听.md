```
// 当前页绑定事件处理函数
window.onpopstate = function (event) {
    console.log(event);
    // window.location.replace(url);
    // 此时可以指定一个别的页面进行跳转。例如跳转到个人中心页面。
    // 建议1：使用window.location.replace(url)，否则回退两次就又回到当前页面了。
    // 然后第三次回退再触发事件跳转，第四次回退又回到当前页面，一直无限循环。
    // 使用replace，会从历史记录里替换掉当前页(当前页丢失，无法通过回退回到当前页)，跳转的那个页面在历史记录里会变成最新的。
    // 这样，回退就会直接回到上上个页面。丢失当前页面的体验。总比回退两次又回到当前页面然后无限循环体验要好。
    // 在建议1的基础上进行建议2：跳转时，跳往下一个页面，别跳往上一个页面。否则，回退一次是上个页面，再回退一次还是上一个页面。
    // 总结：如果不使用replace，回退会导致两个页面之间进行无限循环跳转。使用了replace之后，尽量往前跳一个新页面，不要往后跳老页面，否则体验会很怪异。
};
// 添加并激活一个历史记录条目。如果不加这些，触发不了上面的事件。
history.pushState({state1: 'state data 1'}, 'title1', '#url1');
history.pushState({state2: 'state data 2'}, 'title2', '#url2');
history.pushState({state3: 'state data 3'}, 'title3', '#url3');
```
