# 常见内联块元素
> MDN 把这几个元素归类到了内联元素中，但是在谷歌浏览器上他们的默认属性是display:inline-block。
* input
* textarea
* select
* button

# 块元素的特性
* 支持所有CSS样式。
* 继承父级宽度。
* 独占一行。
* 代码换行不被解析。会忽略标签内的首尾空格。标签内多个连续的非首尾空格会被解析成一个空格。

# 内联元素的特性
* 不支持宽高，不支持上下margin，不支持上下padding。【img、canvas、video、audio标签除外】
* 内容撑开宽度。
* 同属性标签同行排列。
* 代码换行被解析为一个空格。不会忽略标签内的首尾空格。标签内多个连续的空格会被解析成一个空格。
* 标签之间的多个换行或多个空格，会被解析为一个空格。空格的大小取决于父级的font-size以及父级的font-family。

# 内联块元素的特性
* 支持所有CSS样式。
* 内容撑开宽度。
* 同属性标签同行排列。
* 代码换行被解析为一个空格。会忽略标签内的首尾空格。标签内多个连续的非首尾空格会被解析成一个空格。
* 标签之间的多个换行或多个空格，会被解析为一个空格。空格的大小取决于父级的font-size以及父级的font-family。

# 我个人对img标签的简介
* 规范上是把img标签归类到内联标签的。用Chrome调试时，img标签的display默认值也是inline。
* 但是浏览器在赋予img标签特性的时候，应是按照内联块的特性赋予的。

## Chrome 135及之后 内联元素的渲染机制发生了变化
```html
<body style="font-size: 40px;line-height: 1;text-align: center;margin: 40px;width: 200px;color: #ffffff;">
  <div style="background: green;"><span style="font-size: 20px;">你好</span></div>
  <!--
  上述div
  Chrome 134及之前 div的高度是20px
  Chrome 135及之后 div的高度是40px 且span居底对齐 且span的高度是29px
  -->
  <div style="background: blue;"><p style="font-size: 20px;margin: 0;">你好</p></div>
  <!--
  上述div
  Chrome 134及之前 div的高度是20px
  Chrome 135及之后 div的高度是20px
  -->
</body>
<!--
Chrome 135及之后 内联元素的渲染机制发生了变化
核心变更：line-height 从 “子元素按自身字体大小算比例” 改为 “直接继承父级计算后的固定值”（如父级 40px 字体 + line-height:1，子元素直接继承 40px 行高）
次要调整1：基线对齐基准统一变更为em 框底部
次要调整2：line-height:normal 固定为 1.45 倍字体大小
-->
```
