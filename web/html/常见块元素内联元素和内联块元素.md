# 常见内联块元素

* button

# 块元素的特性
* 支持所有CSS样式。
* 继承父级宽度。
* 独占一行。
* 代码换行不被解析。会忽略标签内的首尾空格。标签内多个连续的非首尾空格会被解析成一个空格。

# 内联元素的特性
* 不支持宽高，不支持上下margin，不支持上下padding（内联元素视觉上支持上下padding但不会撑开父级的高度）。【img、canvas、video、audio标签除外】
* 内容撑开宽度。
* 同属性标签同行排列。
* 代码换行被解析为一个空格。不会忽略标签内的首尾空格。标签内多个连续的空格会被解析成一个空格。
* 标签之间的多个换行或多个空格，会被解析为一个空格。空格的大小取决于父级的font-size以及父级的font-family。

# 内联块元素的特性
* 支持所有CSS样式。
* 内容撑开宽度。
* 同属性标签同行排列。
* 代码换行被解析为一个空格。会忽略标签内的首尾空格。标签内多个连续的非首尾空格会被解析成一个空格。
* 标签之间的多个换行或多个空格，会被解析为一个空格。空格的大小取决于父级的font-size以及父级的font-family。

# 我个人对img标签的简介
* 规范上是把img标签归类到内联标签的。用Chrome调试时，img标签的display默认值也是inline。
* 但是浏览器在赋予img标签特性的时候，应是按照内联块的特性赋予的。

## Chrome 135及之后 块元素的行盒高度计算优先级进行了调整 且仅针对内部包含内联/内联块元素的场景
```html
<body style="font-size: 40px;line-height: 1;margin: 40px;width: 80px;color: #ffffff;">
  <div style="background: blue;"><p style="font-size: 20px;margin: 0;">你好</p></div>
  <!--
  上述div
  Chrome 134及之前 div的高度是20px p的高度是20px
  Chrome 135及之后 div的高度是20px p的高度是20px
  -->
  <div style="background: green;"><span style="font-size: 20px;">你好</span></div>
  <!--
  上述div
  Chrome 134及之前 div的高度是20px span的高度是20px
  Chrome 135及之后 div的高度是40px span的高度是29px span居底对齐
    因Chrome 135及之后 块元素的行盒高度计算优先级进行了调整 且仅针对内部包含内联/内联块元素的场景
    div的高度是40px 是因为div继承body计算后的固定行高40px 且span行高29px<40px div优先取自身固定行高
      块元素包含内联/内联块子元素时 行盒高度优先级为自身继承或设置的line-height固定值
      若自身line-height固定值小于子元素的实际视觉行高 父级行盒高度会被子元素实际行高撑开 等于子元素实际行高
    span的高度是29px 是因为20px字体大小乘以中文字体固有行高比例≈1.45 与继承的40px行高无关
      内联元素的视觉高度由字体固有行高决定 line-height仅控制行间距
      字体固有行高 = 字体大小 * 对应字体的默认行高比例 中文字体的默认行高比例约为1.45
    span居底对齐 是因为span基线统一锚定em框底部 div行盒高度40px>span视觉高度29px span沿基线贴底显示
      内联元素的基线统一锚定到em框底部 不再由字体文件自定义基线位置
  建议使用flex布局规避上述细节问题
  -->
</body>
```
